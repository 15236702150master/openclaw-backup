# 记忆记录规范

**目的**: 确保重要信息不丢失，形成持续积累的知识库

---

## 📋 记忆分级

### L1 - 立即记录（完成任务后立即）
**触发条件**:
- ✅ 完成重大技术突破
- ✅ 学会新技能/工具
- ✅ 解决棘手问题
- ✅ 用户明确说"记住这个"

**记录位置**: `MEMORY.md`

**示例**:
- TuriX-CUA Linux 部署（2026-03-06）
- 微信公众号文章读取方法
- API 密钥配置

### L2 - 每日摘要（23:00 自动）
**触发条件**:
- 每天 23:00 心跳检查时
- 由 daily-summary-ai 自动生成

**记录位置**: `memory/YYYY-MM-DD.md`

**内容**:
- 当天完成的任务
- 学到的知识点
- 遇到的问题和解决方案

### L3 - 每周审查（周日手动）
**触发条件**:
- 每周日晚上
- 审查 daily files，提炼到 MEMORY.md

**操作**:
1. 阅读本周 `memory/*.md` 文件
2. 识别重要信息
3. 更新 MEMORY.md
4. 删除临时文件

## 📝 记录格式

### MEMORY.md 条目模板

```markdown
## 项目名称 ⭐ 重要性标记（2026-03-06）

### 描述
- **功能**: 一句话说明
- **位置**: `/path/to/project`
- **状态**: ✅ 已部署 / ⏳ 进行中 / ❌ 已废弃

### 关键配置
```bash
# 重要命令或配置
```

### 测试结果
- ✅ 功能 A 正常
- ⚠️ 功能 B 待完善

### 使用方法
```bash
# 快速启动命令
```

### 重要文件
- 配置文件：`/path/to/config`
- 文档：`/path/to/doc.md`

### 关键教训
1. 教训一
2. 教训二

### 下一步
- [ ] 待办事项

---
**记录时间**: 2026-03-06  
**意义**: 为什么这个重要
```

## 🔄 记忆维护

### 定期清理
- **每天**: 23:00 自动生成 daily log
- **每周**: 审查 daily logs，提炼到 MEMORY.md
- **每月**: 清理过时的临时文件

### 去重规则
写入前检查：
1. 是否已存在相同信息？
2. 是否是临时信息（不需要长期记忆）？
3. 是否可以在外部文档找到？

### 更新策略
- **新增**: 直接添加到对应章节
- **修改**: 保留历史，添加新版本并标注日期
- **删除**: 标记为"已废弃"，不直接删除

## 💡 记忆技巧

### 重要性标记
- ⭐⭐⭐ 核心能力（必须记住）
- ⭐⭐ 重要技能（经常使用）
- ⭐ 一般信息（需要时查阅）

### 关键词索引
在条目末尾添加：
```
**标签**: #TuriX #ComputerUse #Linux #自动化
```

### 交叉引用
```
参见：[[TuriX-CUA]] 或 参考：/path/to/doc.md
```

## 🚨 常见错误

### ❌ 错误做法
- 等到用户提醒才记录
- 只记录成功，不记录失败教训
- 记录太详细，变成日志
- 记录太简略，无法复用

### ✅ 正确做法
- 完成任务后立即记录
- 成功和失败都记录
- 记录关键信息，细节放外部文档
- 包含使用方法、配置、教训

## 📊 记忆质量检查

每次记录后问自己：

2. **别人能根据这个复现吗？**
3. **这个信息值得长期保存吗？**
4. **有没有更简洁的表达方式？**
