# 任务分工架构 - AI vs 脚本

## 核心原则

```
需要理解、判断、推理的任务 → AI 模型
机械性、规则明确、重复性的任务 → 脚本
```

---

## AI 模型负责的任务

### 1. 每日摘要生成 ⭐

**为什么用 AI：**
- 需要理解对话内容
- 判断信息重要性
- 语义去重
- 准确分类

**实现方式：**
```
对话历史 → AI 分析 → 提取关键事项 → 用户确认 → AI 分类记录
```

**Token 消耗：** ~2k-5k tokens/次（值得）

**脚本位置：** `scripts/daily-summary-ai.py`

---

### 2. 记忆提炼（每周）

**为什么用 AI：**
- 判断信息的长期价值
- 识别关联信息
- 提炼核心内容

**实现方式：**
```
memory/日志 → AI 审查 → 提炼重要内容 → 更新 MEMORY.md
```

**Token 消耗：** ~3k-8k tokens/周

---

### 3. 待办智能提醒

**为什么用 AI：**
- 理解任务上下文
- 智能建议提醒时间
- 识别任务优先级

**实现方式：**
```
用户：明天有会议
AI：好的，几点开始？需要提前多久提醒？
用户：下午 3 点，提前 15 分钟
AI：已设置 14:45 提醒
```

**Token 消耗：** ~200-500 tokens/次

---

### 4. 对话摘要（长对话）

**为什么用 AI：**
- 理解对话主题
- 提取关键决策
- 压缩历史记录

**实现方式：**
```
对话轮次 >20 → AI 总结 → 保存摘要 → 清理历史
```

**Token 消耗：** ~1k-3k tokens/次

**节省：** 用 500 tokens 替代 10k tokens 历史

---

### 5. 情感分析（可选）

**为什么用 AI：**
- 理解用户情绪
- 调整回复语气
- 识别不满/困惑

**实现方式：**
```
用户消息 → AI 分析情感 → 调整回复策略
```

**Token 消耗：** ~100-300 tokens/次

---

## 脚本负责的任务

### 1. 文件清理 ✅

**为什么用脚本：**
- 规则明确（时间、大小）
- 不需要理解内容
- 机械性操作

**实现方式：**
```bash
find /tmp -type f -mtime +1 -delete
```

**Token 消耗：** 0

**脚本位置：** `scripts/cleanup.sh`

---

### 2. 日志轮转 ✅

**为什么用脚本：**
- 固定流程
- 时间判断
- 文件操作

**实现方式：**
```bash
find memory/ -name "*.md" -mtime +30 -exec mv {} archive/ \;
```

**Token 消耗：** 0

---

### 3. 状态检查 ✅

**为什么用脚本：**
- 简单判断
- 数据读取
- 报告生成

**实现方式：**
```python
if task["nextRun"] < now:
    print("任务已错过")
```

**Token 消耗：** 0

**脚本位置：** `scripts/task-manager.py`

---

### 4. 备份压缩 ✅

**为什么用脚本：**
- 机械操作
- 工具调用
- 不需要理解

**实现方式：**
```bash
tar -czf archive.tar.gz old_logs/
```

**Token 消耗：** 0

---

### 5. 健康检查 ✅

**为什么用脚本：**
- 指标收集
- 阈值比较
- 报告生成

**实现方式：**
```python
size = get_workspace_size()
if size > 500 * 1024 * 1024:
    alert("工作区过大")
```

**Token 消耗：** 0

| 状态检查 | 0 | 0 | 脚本完全胜任 |

## 最佳实践

### ✅ 正确使用 AI

```
1. 需要理解语义 → 用 AI
2. 需要判断重要性 → 用 AI
3. 需要分类/归纳 → 用 AI
4. 需要创造性 → 用 AI
5. 需要与人交流 → 用 AI
```

### ✅ 正确使用脚本

```
1. 文件操作 → 用脚本
2. 时间判断 → 用脚本
3. 规则执行 → 用脚本
4. 数据收集 → 用脚本
5. 批量处理 → 用脚本
```

### ❌ 避免的误区

```
❌ 用脚本做语义分析（不准确）
❌ 用 AI 做文件清理（浪费 token）
❌ 用脚本判断重要性（不可靠）
❌ 用 AI 做简单数学计算（过度）
```

## 实际案例

### 案例 1：每日摘要

**脚本方案（不推荐）：**
```python
# 关键词匹配
if "安装" in message:
    record(message)
# 问题：无法理解上下文，可能遗漏或误判
```

**AI 方案（推荐）：**
```python
# 语义理解
analysis = ai_analyze(conversations)
# 优势：理解"装了个新技能" = 安装，理解上下文
```

---

### 案例 2：文件清理

**AI 方案（不推荐）：**
```python
# 浪费 token
ai_decide_which_files_to_delete()
# 问题：文件清理不需要理解，规则明确
```

**脚本方案（推荐）：**
```bash
# 规则明确
find /tmp -name "*.tmp" -mtime +1 -delete
# 优势：快速、准确、0 token
```
